# Book Digest

A SwiftUI iOS app for generating and listening to audio digests of productivity and self-help books.

Digests and their audio are generated **once** and shared between all users through a Supabase backend. Users bring their own API keys: the app sends them with a generation request to a Supabase edge function, which calls OpenAI (digest text) and ElevenLabs (TTS audio) and stores the results — keys are used in-memory only and never stored server-side. Once a book has been generated by anyone, every other user can read and play it for free.

The app includes:

- A visual book picker with stylized cover art.
- Shared digest generation via a Supabase edge function (OpenAI Responses API, default `gpt-5.5`).
- Shared audio with a single fixed ElevenLabs voice, downloaded once and played offline.
- Playback with pause, resume, scrubbing, lock-screen controls, and CarPlay.
- Settings for local OpenAI and ElevenLabs API keys.

## iOS app

Open `BookDigest.xcodeproj` in Xcode or build the `BookDigest` scheme for an iOS simulator. The Supabase Swift SDK is fetched automatically via Swift Package Manager.

Before first run, provide your Supabase project's host and publishable key
(`sb_publishable_...`, from Dashboard → Project Settings → API Keys):

```sh
cp Config/Secrets.example.xcconfig Config/Secrets.xcconfig
# then edit Config/Secrets.xcconfig with your values
```

`Config/Secrets.xcconfig` is gitignored, so your values are never committed.
The values are injected into the app at build time via Info.plist and read by
`BookDigest/Services/Supabase/SupabaseConfig.swift`.

## Backend (`supabase/`)

- `migrations/` — `digests` table (text + audio status per book, RLS read-only for clients), `generation_requests` (rate limiting), atomic claim RPCs, and the private `digest-audio` storage bucket.
- `functions/generate-digest/` — the edge function that claims a book, generates text/audio with the requester's keys, and stores the result. Voice and speech model are fixed constants in the function.

The project uses Supabase's new API keys (`sb_publishable_...` / `sb_secret_...`), not the legacy anon/service_role JWTs. Because the functions gateway can only verify legacy JWT keys, `verify_jwt` is disabled in `config.toml` and the function authenticates callers itself via `auth.getUser()`.

Local development:

```sh
supabase start                                      # local stack, applies migrations
supabase functions serve generate-digest            # run the function locally
```

Deploy to the cloud project:

```sh
supabase link --project-ref <your-project-ref>
supabase db push
supabase functions deploy generate-digest
```

Then in the Supabase dashboard enable **Auth → Sign in / Providers → Anonymous sign-ins** (the app signs users in anonymously for rate limiting).

Digest prompts and the text sanitizer exist in both Swift and TypeScript; if you change one, mirror it in the other (`supabase/functions/_shared/prompts.ts` / `sanitize.ts`).
